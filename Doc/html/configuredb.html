3.2. Pozostałe opcje konfiguracyjne

Większość parametrów możesz ustawić bezpośrednio z poziomu przeglądarki. Wyjątkiem jest tutaj konfiguracja portów wyjściowych (przypisanie wyjść do określonych urządzeń), które należy wykonać bezpośrednio w bazie.

Aby zmienić przypisanie wyjścia do urządzenia, wykonaj następujące kroki: 

root@raspberrypi:~# mysql -u aquapi -p 
Enter password:
Welcome to the MySQL monitor.  Commands end with ; or \g.
mysql> USE aquapi;
Database changed
mysql> select * from devices;

Teraz możesz zmienić przypisanie wyjścia do urządzenia. Zanim jednak to zrobisz, zapoznaj się z możliwymi ustawieniami:

  • disabled - urządzenie wyłączone. Aplikacja nie będzie używać tego urządzenia.

  • rpi:gpio:? - urządzenie będzie korzystać z portu GPIO. Zamiast ? należy podać numer portu.

    Podpowiedź

    Przykładowy porty to rpi:gpio:4 - oznacza on pin nr. 4 na porcie GPIO według rozpiski WiringPi

  • relbrd:ttyUSB?:? - urządzenie będzie korzystać z karty RelayBoard. Zamiast pierwszego ? należy podać numer portu USB do którego została podłączona karta. Zamiast drugiego ? należy podać numer przekaźnika na karcie.

    Podpowiedź

    Przykładowy porty to relbrd:ttyUSB0:4 - oznacza on przekaźnik nr. 4 na karcie RelayBoard widoczej w systemie pod /dev/ttyUSB0

  • dummy - urządzenie dummy, przydatne przy rozwoju oprogramowania. Aplikacja normalnie korzysta z tego urządzenia, ale zmiana jego stanu nie ma na nic wpływu.

Teraz jeśli wiesz jakie jest nazewnictwo portów, możesz przypisać takie porty jakie sobie zażyczysz: 

mysql> update devices set output="rpi:gpio:4" where device="heater";
Query OK, 1 row affected (0.00 sec)
mysql> update devices set output="relbrd:ttyUSB0:4" where device="light";
Query OK, 1 row affected (0.00 sec)

Powyższe spowoduje że AquaPi będzie sterowało grzałką podłączoną pod pin 4 GPIO, oraz oświetleniem podłączonym pod 4 przekaźnik karty RelayBoard wpiętej w ttyUSB0.
